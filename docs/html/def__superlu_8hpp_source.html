63 // We also need superlu_malloc() and superlu_free().
64 // When using the original SuperLU code directly, you (the user) may
65 // define USER_MALLOC and USER_FREE, but the joke is on you because
66 // if you are linking against SuperLU and not compiling from scratch,
67 // it won't actually make a difference anyway! If you've compiled
68 // SuperLU against a custom USER_MALLOC and USER_FREE, you're probably up
69 // shit creek about a thousand different ways before you even get to this
70 // code, so, don't do that!
